# C360客户全景平台 - 业务逻辑闭环与模块审查报告 (H5端专区)

基于《Customer360项目需求规格说明书》及目前 H5 端的模块开发情况，以下是专属 **H5前端 (WeCom/BDC 嵌入端)** 的业务未闭环断点清单：

---

## 🛑 【核心】业务未闭环断点清单

### 1️⃣ 沟通记录与跟进留痕：销售触达后动作无法回写
- **严重级别**：🔴 **[最高 / Critical]**
- **当前状态**：H5 的 `CommunicationRecords.vue` 只有单向的只读展示时间轴。
- **未闭环痛点**：缺乏用户录入“写”数据的业务闭环。一线销售或 BDC 呼叫/面谈客户后，无法在前端录入“每一次的沟通结果（如：首次电访、接通且高意向客户、空号）”。导致任何商机的生命周期只能长期停摆，业务成果和客户意向无法回溯至平台。
- **📄 对应 PRD 需求文档条款**：
  > **4.9.4. 商机生命周期与结果闭环功能**：商机跟进结果由BDC和DMS等源数据同步至本系统；系统完整记录商机从生成到闭环的全流程数据。
  > **2.2.1. 商机生成 / 3.4. 自动化商机生成**：商机由C360自动生成后推送给BDC跟进，根据返场具体项目闭环商机结果。
- **填补建议 (CRUD)**：
  - **增 (C)**：H5端必须明确补充【新增跟进活动记录】表单区域或全局悬浮按钮，要求必填“沟通方式”、“沟通结果枚举（继续跟进/转化/战败）”及“跟进备注”。
  - **改 (U)**：坐席提交后，由前端发请求必定连带更新该条客户身上相关商机或任务的 `Status`（完成事件闭环）。

### 2️⃣ 顾客视图与数据回流机制：一线纠错无操作接口
- **严重级别**：🟠 **[高 / High]**
- **当前状态**：H5 的数据档案面板绝大多数为只读看板。
- **未闭环痛点**：PRD 要求必须要为 BDC 客服留有“赋予和剥除手工标签”、“提交联系方式错误或用户死亡等客户异常”的业务功能。由于前后端目前不具备完善的双向表单体系，系统缺乏获取数据一线人员反馈机制的反哺能力。
- **📄 对应 PRD 需求文档条款**：
  > **4.11. BDC嵌入页和Wecom嵌入应用**：数据交互：支持 BDC/WeCom 端数据回传（标签添加或删除、顾客信息准确性反馈）。
  > **4.7.3. 标签反馈与更正流程**：允许用户通过前端对识别出的标签不一致或错误提交反馈 / 更正。
- **填补建议 (CRUD)**：
  - **增 (C)**：H5 端增加全局【纠错与修改反馈提交】快捷方式入口，只要提交，此记录必须自动进入 PC 端的 `errorCorrection`（异常控制中心）待处理待办流转。
  - **增/删 (C/D)**：需全面放开支持【手工属性标签】的池内下拉拉取点击赋予或剥夺权限。

### 3️⃣ 车辆全生命周期管理：视图组件完全空白
- **严重级别**：🔴 **[最高 / Critical]**
- **当前状态**：`src/views/VehicleInfo.vue` 文件已创建但大小为 0 字节（内容完全空白），未实现任何 UI 或交互。
- **未闭环痛点**：销售与售后人员完全无法在 H5 端查看客户名下的车辆详情、合同信息及全生命周期状态。
- **📄 对应 PRD 需求文档条款**：
  > **2.3 车辆全生命周期 (Vehicle Life Cycle — Full List)**：记录车辆从销售线索、合同签署到售后保有全过程的精细字段。
- **填补建议 (CRUD)**：
  - **查 (R)**：尽快完成 `VehicleInfo.vue` 视图的页面搭建，对接后端的车辆详情接口。
  - **改 (U)**：预留对车辆运行状态（或解绑操作）的反馈入口。

### 4️⃣ 资产与卡券精细化模块：前端页面缺失
- **严重级别**：🟠 **[高 / High]**
- **当前状态**：项目中缺乏名为 `Asset.vue` 或 `Coupon.vue` 等承载权益数据的独立组件或路由。仅在 Mock 数据和 TS 类型中有定义。
- **未闭环痛点**：无法透视客户具体的代金券、优惠券及赠送礼包，阻碍售后 SA 推销核销卡券形成业务营收。
- **📄 对应 PRD 需求文档条款**：
  > **2.4 资产与卡券精细化 (Assets & Coupons — Full List)**：覆盖所有营销权益、售后返券及购车礼包（15 项精细化字段）。
- **填补建议 (CRUD)**：
  - **查 (R)**：新增专门的卡券列表弹窗或 Tab 视图，展示优惠券的面额、状态、有效期及核销二维码/核销码。

### 5️⃣ 公司账户经办人管理：多联系人维护无接口
- **严重级别**：🟠 **[高 / High]**
- **当前状态**：`Home.vue` 中能展示基础的经办人/联系人信息，但缺乏经办人的 CRUD（增删改查）管理入口。
- **未闭环痛点**：B 端企业客户的人员变动频繁，销售在拜访时如发现企业联系人变更，无法在手机端实时更新系统记录，导致下次联系依然找错人。
- **📄 对应 PRD 需求文档条款**：
  > **2.8 公司账户经办人管理 (B-Side Handler Management)**：针对公司账户，需独立维护多位联系人信息（包含角色、优先级管理）。
- **填补建议 (CRUD)**：
  - **增/改/删 (C/U/D)**：当 H5 处于 `Company` 视图且拥有权限时，允许业务员通过弹窗管理（新增/停用）该企业的经办人、送修人和主要联系人。

### 6️⃣ 智能商机预警：缺少业务处理状态回写闭环
- **严重级别**：� **[最高 / Critical]**
- **当前状态**：首页可能存在商机的简单展示（UI 层），但对商机的“处理状态”流转无任何操作闭环。
- **未闭环痛点**：BDC 面板上虽然能看到系统推送来的红/黄/蓝高优商机，但处理完（如：已打电话跟进）后，在此 H5 上无法一键将状态变更为“已完成”或“已失效”。
- **�📄 对应 PRD 需求文档条款**：
  > **2.2 智能商机预警 (Smart Opportunity — Tech Detail)**：处理状态 [待处理, 处理中, 已推送, 已完成, 已失效]... 支持全链路状态追踪。
- **填补建议 (CRUD)**：
  - **改 (U)**：在商机卡片上提供明显的【处理】或【跟进】按钮。关联全局更新动作，将其状态回写至系统流转大盘。
