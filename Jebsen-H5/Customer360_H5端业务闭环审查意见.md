# C360客户全景平台 - 业务逻辑闭环与模块审查报告 (H5端专区)

基于《Customer360项目需求规格说明书》及目前 H5 端的模块开发情况，以下是专属 **H5前端 (WeCom/BDC 嵌入端)** 的业务未闭环断点清单：

---

## 🛑 【核心】业务未闭环断点清单

### 1️⃣ 沟通记录与跟进留痕：销售触达后动作无法回写
- **严重级别**：🔴 **[最高 / Critical]**
- **当前状态**：H5 的 `CommunicationRecords.vue` 只有单向的只读展示时间轴。
- **未闭环痛点**：缺乏用户录入“写”数据的业务闭环。一线销售或 BDC 呼叫/面谈客户后，无法在前端录入“每一次的沟通结果（如：首次电访、接通且高意向客户、空号）”。导致任何商机的生命周期只能长期停摆，业务成果和客户意向无法回溯至平台。
- **📄 对应 PRD 需求文档条款**：
  > **4.9.4. 商机生命周期与结果闭环功能**：商机跟进结果由BDC和DMS等源数据同步至本系统；系统完整记录商机从生成到闭环的全流程数据。
  > **2.2.1. 商机生成 / 3.4. 自动化商机生成**：商机由C360自动生成后推送给BDC跟进，根据返场具体项目闭环商机结果。
- **填补建议 (CRUD)**：
  - **增 (C)**：H5端必须明确补充【新增跟进活动记录】表单区域或全局悬浮按钮，要求必填“沟通方式”、“沟通结果枚举（继续跟进/转化/战败）”及“跟进备注”。
  - **改 (U)**：坐席提交后，由前端发请求必定连带更新该条客户身上相关商机或任务的 `Status`（完成事件闭环）。

### 2️⃣ 顾客视图与数据回流机制：一线纠错无操作接口
- **严重级别**：🟠 **[高 / High]**
- **当前状态**：H5 的数据档案面板绝大多数为只读看板。
- **未闭环痛点**：PRD 要求必须要为 BDC 客服留有“赋予和剥除手工标签”、“提交联系方式错误或用户死亡等客户异常”的业务功能。由于前后端目前不具备完善的双向表单体系，系统缺乏获取数据一线人员反馈机制的反哺能力。
- **📄 对应 PRD 需求文档条款**：
  > **4.11. BDC嵌入页和Wecom嵌入应用**：数据交互：支持 BDC/WeCom 端数据回传（标签添加或删除、顾客信息准确性反馈）。
  > **4.7.3. 标签反馈与更正流程**：允许用户通过前端对识别出的标签不一致或错误提交反馈 / 更正。
- **填补建议 (CRUD)**：
  - **增 (C)**：H5 端增加全局【纠错与修改反馈提交】快捷方式入口，只要提交，此记录必须自动进入 PC 端的 `errorCorrection`（异常控制中心）待处理待办流转。
  - **增/删 (C/D)**：需全面放开支持【手工属性标签】的池内下拉拉取点击赋予或剥夺权限。

## 📄 用户界面逐页分析矩阵 (Page-by-Page Analysis)

基于各页面实际代码文件，提炼其实际承载职责与缺漏建议摘要：

| 目录/页面视图 | 页面主要职责与当前承载能力 | 业务完整度分析与改进建议 |
| :--- | :--- | :--- |
| `Home.vue` | **客户360画像综合首屏**。承载多 OneID 切换、基本属性卡片展示、标签全览及各类服务入口。 | **基本盘完好，双向性不足**。标签只读不可手工赋删。需加入快速发起工单/指正错误属性的浮窗入口给到 BDC 坐席。 |
| `CommunicationRecords.vue` | **客患跟进沟通流水记录**。惰性加载展示销售沟通的方式、结果、操作人员和详情备注。 | **缺失最核心的闭环填写控制**。只读面板必须配套添加按钮，允许跟进人直接在 H5 页面写入外呼/线索推延反馈。 |
| `FinancialLoan.vue` | **金融分期记录清单**。展示贷款金融机构、额度、费率贴息细节及还款状态。 | **已闭环（只读展示即可）**。此类数据多由后端同步拉取生成，前端展示清晰合规即可。 |
| `Maintenance.vue` | **保险工单记录清单**。展示保单号、各类商业及交强险承保公司、期限。 | **已闭环（只读展示即可）**。符合档案看板诉求。 |
| `MaintenanceRecords.vue` | **具体维保消费台账**。涵盖机修、钣喷等类型以及总花费、所去门店。 | **已闭环（只读展示即可）**。符合跨门店数据共享及看板诉求。 |
| `MarketingCampaigns.vue` | **线下市场推广活动参与清单**。展示商企端组织的各项市场宣发记录（签到、核销）。 | **已闭环（只读展示即可）**。数据由主端推送下发，作为画像标签及客史追踪依据。 |
