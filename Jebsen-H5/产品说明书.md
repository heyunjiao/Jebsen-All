## 二、项目概述

|------|------|

## 三、鉴权与入口

### 3.1 运行环境识别

| 环境 | 识别方式 | 鉴权方式 |
|------|----------|----------|
| **微信** | User-Agent 含 `wxwork` | OAuth2 静默登录：无 code 时跳转微信授权页，带 code 回跳后请求后端 `/auth/wechat/token` 换取 token，写入 localStorage。 |
| **内部系统** | URL 存在 `token` 或 `ticket` 参数 | 前端解码 token（支持 JWT 或 Base64 JSON），解析出 `token` 与 `userId`，写入 localStorage，并清除 URL 上的 token 参数。 |
| **未知** | 上述均不满足 | 无自动鉴权，依赖后续接口 401 或业务逻辑提示。 |

**环境变量**（与鉴权相关）：

- `VITE_WECHAT_APP_ID`：微信 AppID
- `VITE_WECHAT_REDIRECT_URI`：微信 OAuth 回调地址
- `VITE_API_BASE_URL`：接口基地址
- `VITE_USE_MOCK`：是否启用 Mock（开发环境可设为 true）

### 3.2 手机号搜索入口

| 项目 | 说明 |
|------|------|
| **触发** | 访问 URL 带参数 `phone`（11 位手机号），例如 `?token=xxx&phone=13800138000`。 |
| **流程** | 1）先按 3.1 完成鉴权；2）调用接口 `getOneIdsByPhone(phone)` 获取该手机号关联的 OneID 列表；3）根据列表条数展示不同 UI。 |
| **OneID 数量与展示** | 见下表。 |

| OneID 数量 | 展示方式 | 说明 |
|------------|----------|------|
| 0 | 全屏空状态 | 提示「未找到关联的客户身份」；按钮「查看本次搜索手机号」可查看当前 URL 中的 phone（Toast 展示）。 |
| 1 | 无切换栏 | 直接展示该唯一身份的画像与数据，不显示「当前身份」栏。 |
| 2 | 顶部横向 Tab | 两个 Tab 并排，点击切换身份；下方数据随所选 OneID 变化。 |
| 3～4 | 顶部横向 Tab | 同上，Tab 可横向滚动。 |
| 5 个及以上 | 顶部下拉 | 显示「当前身份」+ 金色徽标「共 N 个 OneID」；点击当前身份弹出 ActionSheet 选择，切换后下方数据联动。 |

**Mock 测试**：`TEST_CASES.md` 中提供多组演示 URL（含 token 与 phone），用于验证多 OneID、空状态、单 OneID 等场景。

### 3.3 个人/公司切换（非手机号入口）

| 项目 | 说明 |
|------|------|
| **触发** | URL 带 token 但不带 `phone` 参数。 |
| **UI** | 右侧悬浮球（可拖拽），文案「看公司」/「回个人」；点击在个人身份与公司身份之间切换。 |
| **数据** | 切换后画像与各区块数据（车辆、维保、商机等）随当前身份（个人 OneID / 公司 OneID）切换。 |

## 四、客户画像主页面（Home.vue）结构

主页面为单页，按「区块 + 弹窗」组织；所有数据随**当前选中的 OneID**（个人/公司或多 OneID 下拉所选）变化。

### 4.1 顶部状态与身份层

| 区域 | 显示条件 | 功能说明 |
|------|----------|----------|
| **OneID 切换栏** | 手机号搜索且 OneID 数量 ≥ 2 | 见 3.2；Tab 或下拉切换后，整页数据刷新为该 OneID。 |
| **加载中** | 手机号搜索且 OneID 列表请求中 | 全屏「正在查询该手机号关联身份...」。 |
| **空状态** | 手机号搜索且 OneID 列表为空 | 全屏空状态 + 「查看本次搜索手机号」按钮。 |
| **公司身份层** | 当前为「公司」身份时 | 名称、ONEID、数据状态监控（同步时间、各来源状态）、平台溯源入口、级标签（如 VIP 车主、钻石客户等）；点击标签若为商机类可打开商机弹窗。 |
| **公司冲突/操作提示** | 公司身份且存在冲突或最新操作 | 冲突：NoticeBar「检测到该存在多条记录冲突…」点击打开冲突解决弹窗；操作：NoticeBar 展示最新操作摘要，点击打开操作日志弹窗。 |
| **个人画像头** | 当前为「个人」身份时 | 头像、姓名、经办人/VIP 等标签、同步信息、关系标签（本人、配偶、公司电话等）。 |
| **个人冲突/操作提示** | 个人身份且存在冲突或最新操作 | 同上，点击打开冲突解决或操作日志弹窗。 |

**数据状态监控（公司视图）**：Popover 展示「同步至: 时间」及各数据源（如 DMS、BDC、Voucher、手工等）状态（正常/异常）；业务数据 T+1 同步说明。

### 4.2 核心信息区块（首屏）

| 区块名称 | 展示内容 | 交互与弹窗 |
|----------|----------|------------|
| **车辆信息** | 车辆数量、列表摘要（车型/车牌/状态等） | 「查看更多」→ 车辆信息弹窗；点击某车状态 → 打开车辆弹窗并直接展开该车状态内联编辑。 |
| **电话信息** | 主号码及关系标签；多号码时展示多条 | 「管理电话号码」→ 电话号码管理弹窗（新增/编辑/删除/合并）。 |
| **画像标签 / 画像标签** | 当前客户已选标签 | 「编辑标签」→ 标签管理弹窗（按分类展示标签池，增删标签）。 |
| **预约信息** | 预约时间、门店、描述等 | 仅展示，无独立弹窗（数据来自 getAppointments）。 |

### 4.3 Tab 区：维保 / 保险 / 沟通 / 营销 / 金融

| Tab | 数据来源 | 说明 |
|-----|----------|------|
| 维保记录 | getMaintenanceRecords | 维保记录列表，可更新维保标签、偏好车型标签等。 |
| 保险合同 | getInsuranceRecords | 保险记录列表，支持分页。 |
| 沟通记录 | （与沟通记录相关接口/数据） | 沟通记录展示。 |
| 营销活动 | getMarketingCampaigns | 线下活动/营销记录，支持分页。 |
| 金融贷款 | getFinancialLoanRecords | 金融贷款记录，支持分页。 |

### 4.4 其他入口与区块

| 入口/区块 | 说明 |
|-----------|------|
| 平台溯源 | 点击入口打开平台溯源弹窗（PlatformFlow），展示数据来源流程图（getPlatformSources）。 |
| 商机信息 | 点击商机相关标签或入口打开商机弹窗，列表展示 getOpportunities。 |
| 操作日志 | 点击顶部「最新操作」或入口打开 OperationLogDialog，列表展示 getOperationLogs。 |
| 资产信息 | 从首屏或相关入口打开资产信息弹窗，列表展示 getAssets（如优惠券/代金券）。 |

## 五、弹窗与编辑能力

### 5.1 弹窗列表

| 弹窗 | 组件/实现位置 | 主要功能 |
|------|----------------|----------|
| 冲突处理 | ConflictResolver | 姓名+手机号多源冲突时，用户选择保留/合并策略并提交（submitNameMobileConflict）。 |
| 电话号码管理 | 内嵌 MobileEditor 或等价表单 | 展示 profile.mobile.items；新增、编辑、删除号码项；合并号码（mergeMobileItems）；关系标签、业务标签、车辆标签、主号码等字段。 |
| 操作日志 | OperationLogDialog | 列表展示操作日志，数据来自 getOperationLogs。 |
| 商机信息 | Home 内 van-popup | 列表/分组展示商机，空状态「暂无商机信息」。 |
| 车辆信息 | Home 内 van-popup | 车辆列表；每车可展开「修改车辆状态」内联选择（禁止二次弹窗）；「相关人员」为某车某角色（如车主、送修人）选择关联号码，内联展示不二次弹窗。 |
| 资产信息 | Home 内 van-popup | 资产/优惠券列表，空状态「暂无资产信息」。 |
| 标签管理 | Home 内 van-popup | 按分类展示标签池；已选/未选列表；添加、删除标签（addTag/removeTag/updateTags）；与 profile.tags 同步。 |
| 基础信息编辑 | Home 内 van-popup | 姓名、年龄、手机号、性别、城市、客户类型、变更理由；提交后调用 updateBasicInfo（提交审核）。 |
| 客户标签选择器 | 单选用 | 从标签池选择单个标签添加。 |
| 用户偏好标签选择器 | 多选用 | 从标签池多选（如服务偏好、偏好车型等）。 |
| 平台溯源 | PlatformFlow | 展示平台溯源流程图，数据 getPlatformSources。 |
| 数据状态监控 | Popover | 公司视图下同步时间与各来源状态，非弹窗。 |

### 5.2 车辆信息弹窗细节

| 能力 | 说明 |
|------|------|
| 车辆列表 | 展示车型、车牌、状态、合同、价格与项目等；空状态「暂无车辆信息」。 |
| 修改车辆状态 | 在当前弹窗内内联展开状态选择（如自用、已售、维修中等），公司/个人状态下发文案可映射，禁止再弹一层。 |
| 相关人员 | 固定展示三个角色（如车主、送修人、使用人）；为某车某角色选择「关联号码」时，在当前弹窗内内联展示号码列表选择，不二次弹窗。 |
| API | getVehicles、updateVehicleStatus；车辆角色与号码关联依赖接口或本地状态同步。 |

### 5.3 编辑权限与数据规则

| 规则 | 说明 |
|------|------|
| 全局只读 | 除下表外，画像字段均为只读展示。 |
| 仅手机号可编辑 | 仅手机号支持在 H5 内直接修改并回传（updateMobile 或号码项的新增/编辑/删除/合并）。 |
| 基础信息变更 | 姓名、年龄、性别、城市、客户类型等通过「基础信息编辑」弹窗填写变更理由后提交，走审核流程（updateBasicInfo）。 |
| 冲突展示 | 多源冲突字段以「最优值」展示在主界面，点击可查看全部来源（C360Field）；冲突解决通过 ConflictResolver 提交。 |
| 字段纠错 | 通过 submitFieldCorrection 提交纠错（字段名、当前值、纠错值、备注）。 |

## 九、与需求规格说明书比对检查清单

以下按模块列出可勾选项，便于与 H5 相关合同逐条比对。

### 9.1 鉴权与入口

- [ ] 微信 OAuth2 静默登录与合同一致（AppID、回调地址、scope）
- [ ] 内部系统 URL token 鉴权方式与合同一致（JWT/Base64、参数名 token/ticket）
- [ ] 手机号搜索入口（URL 参数 phone）与合同一致
